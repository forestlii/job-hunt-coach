# 简历优化：从"自传思维"到"广告思维"

## 核心认知
HR 看简历不是要全面了解你，而是在几秒内找"删掉你的理由"。一个模糊的表述、一段与岗位无关的经历、一个没有数据支撑的结果，都可能成为被筛掉的理由。**"全面展示"恰恰输给"精准匹配"。**

所以简历要改的不是排版和措辞，而是**叙事逻辑**：在"我是谁"之上，加上"我能解决什么问题"。

> 好简历的标准：不是文字最漂亮的，而是让 HR 看完想"**打个电话进一步了解**"的。

## 四步法

### 第一步：岗位锚定（一岗一份）
- 一份简历只讲一个故事，一个岗位对应一份定制简历。一份想满足所有岗位的简历，往往精准满足不了任何一个。
- 从 JD 里提取反复出现的核心能力关键词（如"数据分析""用户增长""跨部门协作"），用它们作为整份简历的组织主线。和这些关键词没有直接关系的内容，考虑删掉——信息越聚焦，核心信号越容易被捕捉。
- 进阶：开头用一句话定义核心价值（"X 年用户增长经验，主导过 XX 产品的增长策略"），相当于给自己贴标签，帮 HR 第一秒建立定位。

### 第二步：经历重构（动作 + 数据 + 价值）
- 问题：大多数人的简历是"职责描述"——"负责日常数据报表的制作和分析""参与例会并记录纪要"。这是 JD 里的语言，不是成果。HR 想知道的不是你的日常，而是你带来的**具体价值**。
- 公式：**动作 + 数据 + 价值**
  - ❌ 负责数据分析
  - ✅ 搭建用户留存分析体系，识别关键流失节点，推动留存率从 62% 提升到 71%
- 数据可前置增强冲击：「将 NPS 从 35 提升到 49」比「通过优化流程将 NPS 提升到 49」更有力。
- 没有具体结果的经历，说服力大打折扣。暂时没有精确数字，就在**诚实前提下**给可感知的范围或相对变化——但绝不编数字。

### 第三步：信号强化（过 ATS + 用 JD 原词）
- 很多大公司在人工看简历前，先用招聘系统（ATS）做关键词匹配；关键词没命中，简历根本到不了 HR 眼前。
- 把 JD 的核心关键词**自然嵌入经历描述**，而不是单独堆在技能栏。一个关键词对应一个具体成果案例。
  - JD 强调"跨部门协作" → "与产品、运营、技术三方协作，推动 XX 项目上线，交付周期缩短 30%"。
- 细节：**尽量用 JD 的原始表述**。JD 写"用户增长"，你写"拉新促活"，意思一样，但关键词匹配率会下降。这不是文字游戏，是提高通过率的策略。

### 第四步：特殊经历优化（gap / 频繁跳槽等）
- 控制篇幅：与目标岗位无关的经历、不超过 3 个月的空窗期，不必在简历里体现，降低面试前就被筛掉的风险，细节留到面试再聊。
- 重新定义叙事：较长的 gap 不是"空白期"，可表述为"职业调整期 / 学习期"+ 完成的学习或认证。
  - 示例："2024.01–2024.06　职业调整期：完成 Google 数据分析认证，独立完成 3 个分析项目。"
- gap / 转行更完整的讲法见 `gap-and-pivot.md`。

## 教练动作
- 让用户**贴 JD 原文 + 现有简历 bullet**，你逐条按公式重写，并标出哪里嵌入了 JD 关键词。
- 重写严守诚实：数据必须是真的；没有的数字不替用户编，引导他回忆真实数据或换可验证的表述。
- 给用户过一遍下面的自检清单。

## 简历自检清单
- [ ] 针对这个岗位定制，不是一稿通投？
- [ ] 开头有没有一句话核心定位？
- [ ] 每条经历是"动作+数据+价值"，而非职责罗列？
- [ ] JD 核心关键词有没有自然出现在经历里（且用 JD 原词）？
- [ ] 有没有与岗位无关、可删的内容在稀释重点？
- [ ] 较长 gap 有没有被重新定义为有信息量的表达？
- [ ] 看完这份简历，HR 会不会想打个电话？
